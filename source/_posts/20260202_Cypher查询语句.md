# Cypher 查询语句实践

[toc]

## 1 简介

### 1.1 什么是 Cypher

Cypher 是 **Neo4j 图数据库**官方提供的声明式查询语言，用来描述**图模式（Pattern）**并从图中查询、创建、更新或删除数据。如果你熟悉 SQL，可以把 Cypher 理解为：

> **面向“关系模式”的 SQL，在图数据库世界里的对应物**。

但不同于 SQL 操作的是“表 + 行”，Cypher 操作的核心对象是：

- **节点（Node）**：实体，如人、订单、设备
- **关系（Relationship）**：节点之间的连接，如“认识”“购买”“属于”
- **属性（Property）**：节点或关系上的键值对

### 1.2 Cypher 的核心特点

1. **模式直观、接近自然语言**
   使用 `()-[]->()` 这种 ASCII 图形来描述关系，非常贴近人类对图的理解。
2. **声明式查询**
   只需要描述“想要什么结果”，不必关心“如何遍历图”。
3. **以关系为一等公民**
   关系不仅有方向和类型，也可以拥有属性。
4. **非常适合多跳关系查询**
   如：朋友的朋友、上下游依赖、知识图谱推理等。

## 1 基本使用

### 1.1 基本元素

#### 1.1.1 节点（Node）

```cypher
(n:Person {name: 'Alice'})
```

- `n`：变量名
- `Person`：标签（Label）
- `{}`：属性集合

#### 1.1.2 关系（Relationship）

```cypher
(n)-[:KNOWS]->(m)
```

- `KNOWS`：关系类型
- `->`：方向（方向很重要）

关系也可以带属性：

```cypher
(n)-[:KNOWS {since: 2020}]->(m)
```

### 1.2 常用关键字

| 关键字 | 作用                     |
| ------ | ------------------------ |
| MATCH  | 查询图模式               |
| WHERE  | 过滤条件                 |
| RETURN | 返回结果                 |
| CREATE | 创建节点/关系            |
| MERGE  | 存在则匹配，不存在则创建 |
| SET    | 更新属性                 |
| DELETE | 删除                     |

### 1.3 最简单的查询

```cypher
MATCH (n)
RETURN n
LIMIT 10
```

含义：返回任意 10 个节点。

## 2 实践中常见场景

### 2.1 查询「实体 → 属性」

> 已知节点（或节点类型），查询它的属性

#### 2.1.1 查询某一类节点的指定属性

```cypher
MATCH (p:Person)
RETURN p.name, p.age
```

#### 2.1.2 根据条件查询属性

```cypher
MATCH (p:Person)
WHERE p.age > 30
RETURN p.name, p.age
```

#### 2.1.3 查询节点的全部属性

使用`properties()`

```cypher
MATCH (p:Person {name: 'Alice'})
RETURN properties(p)
```

#### 2.1.4 查询关系上的属性

```cypher
MATCH (a:Person)-[r:KNOWS]->(b:Person)
RETURN a.name, b.name, r.since
```

### 2.2 查询「属性 → 实体」

> 已知属性条件，反查对应的节点或关系

#### 2.2.1 根据属性定位节点

```cypher
MATCH (p:Person {name: 'Alice'})
RETURN p
```

#### 2.2.2 使用 WHERE 进行模糊 / 组合查询

```cypher
MATCH (p:Person)
WHERE p.name CONTAINS 'Al'
RETURN p
```

或多条件：

```cypher
MATCH (p:Person)
WHERE p.age >= 18 AND p.age <= 30
RETURN p.name, p.age
```

#### 2.2.3 根据关系属性反查实体

```cypher
MATCH (a)-[r:KNOWS]->(b)
WHERE r.since < 2018
RETURN a, b
```

#### 2.2.4 属性 → 多跳实体

查询 Alice 的 1~3 跳朋友

```cypher
MATCH (a:Person {name: 'Alice'})-[:KNOWS*1..3]->(b)
RETURN DISTINCT b
```

## 3 在景图平台中的应用

景图平台基于知识图谱的 **GraphRAG** 方案构建。在涉及图查询的任务中，若仅依赖自然语言或向量检索，往往会出现 **查询不全或命中不准** 的问题。因此，需要结合 **Neo4j 的 Cypher 查询语句**，对图数据进行精确、可控的结构化检索。

整体上，图查询可分为两条典型路线：

1. **实例 → 属性**：根据给定实例名称，查询该实例关联的所有属性或指定属性。
2. **属性 → 实例**：根据一组过滤条件（属性约束），查询所有满足条件的实例。

> 需要注意区分：
>
> - **实例（Instance）**：图中的具体对象，如“X-15 试验机”。
> - **实体/属性节点（Entity / Attribute）**：描述实例特征的节点，如“最大飞行速度”“国家”等。

在实践中，Cypher 查询语句经历了多轮演进与优化，下面按两条路线分别说明。

### 3.1 实例→属性

#### 3.1.1 初始查询方式

```cypher
            USER INPUT: 'X-15试验机的最大飞行速度是多少？'
            QUERY: 
                MATCH (e)-[]-(a1)  
                WHERE e.name CONTAINS "X-15试验机"  
                    AND a1.name CONTAINS "最大飞行速度"
                RETURN a1.name AS name
```

该查询直接返回属性节点 `a1`，能够得到结果，但存在一个问题：

- 程序难以判断**什么时候应该返回实例** `e`**，什么时候应该返回属性** `a1`。

#### 3.1.2 同时返回实例与属性（字符串拼接）

为减少歧义，将实例和属性同时返回，并在查询阶段进行字符串拼接：

```cypher
            USER INPUT: 'X-15试验机的最大飞行速度是多少？'
            QUERY: 
                MATCH (e)-[]-(a1)  
                WHERE e.name CONTAINS "X-15试验机"  
                    AND a1.name CONTAINS "最大飞行速度"
                RETURN DISTINCT
                    e.name + "：" + a1.name AS name
```

这种方式在展示层可用，但**不利于后续构造三元组**（head / relation / tail）。

#### 3.1.3 显式返回三元组结构

为支持后续 GraphRAG 或结构化处理，改为显式返回三元组及节点类型：

```cypher
        MATCH (e)-[r]-(a1)  
        WHERE e.name CONTAINS "X-15试验机"
        RETURN DISTINCT
            e.name AS head,
            type(r) AS relation,
            a1.name AS tail,
            COALESCE(properties(e).schema_type, "Unknown Node") AS head_props,
            COALESCE(properties(a1).schema_type, "Unknown Node") AS tail_props
```

#### 3.1.4 属性节点拆分后的适配

在图结构更新后，属性节点被拆分为：

- `name`：属性名（如“国家”）
- `value`：属性值（如“中国”）

对应查询调整为：

```cypher
       MATCH (e)-[r]-(a1)  
        WHERE e.name CONTAINS "X-15试验机"
        RETURN DISTINCT
            e.name AS head,
            type(r) AS relation,
            a1.name + ":" + a1.value AS tail,
            COALESCE(properties(e).schema_type, "Unknown Node") AS head_props,
            COALESCE(properties(a1).schema_type, "Unknown Node") AS tail_props

```

#### 3.1.5 固定关系类型，避免 text2cypher 错误

在上述查询中，关系变量 `r` 并未参与任何过滤条件，仅用于结果返回。在实际使用中发现：

- **text2cypher 有时不会显式生成关系变量** `r`
- 但在 `RETURN` 中使用 `r` 会直接导致查询报错

为提升稳定性，采取如下策略：

- 不再显式声明关系变量
- 统一返回固定关系类型 `HAS_ATTRIBUTE`

```cypher
    USER INPUT: 'X-15试验机的最大飞行速度是多少？'
        MATCH (e)-[]-(a1)  
        WHERE e.name CONTAINS "X-15试验机"
        RETURN DISTINCT
            e.name AS head,
            'HAS_ATTRIBUTE' AS relation,
            a1.name + ":" + a1.value AS tail,
            COALESCE(properties(e).schema_type, "Unknown Node") AS head_props,
            COALESCE(properties(a1).schema_type, "Unknown Node") AS tail_props
```

### 3.2 属性→实例

#### 3.2.1 初始查询方式

根据查询语句中给出的属性条件数量，分别匹配对应数量的属性节点，并返回满足所有条件的实例。

```cypher
            USER INPUT: '美国的双发涡轮螺旋桨短程支线客机共有多少种型号？具体包括哪些'
            QUERY: 
                MATCH (e)-[]->(a1)  
                MATCH (e)-[]->(a2)  
                MATCH (e)-[]->(a3)  
                WHERE a1.name CONTAINS "双发涡轮螺旋桨短程客机"
                    AND a2.name CONTAINS "美国" 
                    AND a3.name CONTAINS "支线"
                    AND e.schema_type = "装备"  
                    RETURN DISTINCT e.name AS name
```

该方式能够正确返回结果，但查询结构与属性条件数量强绑定，可扩展性较差。

#### 3.2.2 同时返回实例与属性（字符串拼接）

延续 3.1 中的思路，尝试在结果中同时返回实例及命中的属性：

```cypher
           USER INPUT: '美国的双发涡轮螺旋桨短程支线客机共有多少种型号？具体包括哪些'
            QUERY: 
                MATCH (e)-[]->(a1)  
                MATCH (e)-[]->(a2)  
                MATCH (e)-[]->(a3)  
                WHERE a1.name CONTAINS "双发涡轮螺旋桨短程客机"
                    AND a2.name CONTAINS "美国" 
                    AND a3.name CONTAINS "支线"
                    AND e.schema_type = "装备"  
                    RETURN DISTINCT 
                        e.name + "：" + a1.name + "，" + a2.name + "，" + a3.name AS name
   
```

但在实际使用中发现：

- 命中实例数量可能较多
- 返回内容冗余，超过实际需求

因此，最终仍回退为**仅返回实例名称**。

#### 3.2.3 使用 EXISTS 约束属性条件

由于上述写法中包含多个 `MATCH`，在属性节点数量较多时容易产生笛卡尔积。为避免该问题，引入 `EXISTS` 子查询，仅用于判断实例是否满足属性条件。

```cypher
      MATCH (e)-[r]->(a)
        WHERE EXISTS { MATCH (e)-[:HAS_ATTRIBUTE]->(a1) WHERE a1.name CONTAINS "双发涡轮螺旋桨短程客机" }
          AND EXISTS { MATCH (e)-[:HAS_ATTRIBUTE]->(a2) WHERE a2.name CONTAINS "美国" }
          AND EXISTS { MATCH (e)-[:HAS_ATTRIBUTE]->(a3) WHERE a3.name CONTAINS "支线" }
          AND (a.name CONTAINS "双发涡轮螺旋桨短程客机" OR a.name CONTAINS "美国" OR a.name CONTAINS "支线")
        RETURN DISTINCT 
            e.name AS head, type(r) AS relation, a.name AS tail,
            COALESCE(e.schema_type, "Unknown Node") AS head_props,
            COALESCE(a.schema_type, "Unknown Node") AS tail_props
   
```

#### 3.2.4 固定关系类型，统一输出结构

与 3.1 中的经验一致，关系变量在此同样不参与过滤，仅用于返回结果，存在 text2cypher 不稳定的问题。因此：

- 移除关系变量
- 统一返回固定关系类型 `HAS_ATTRIBUTE`
- 使用关键词数组提升可扩展性

```cypher
    USER INPUT: '美国的双发涡轮螺旋桨短程支线客机共有多少种型号？具体包括哪些'
        WITH ["美国", "双发涡轮螺旋桨短程客机", "支线"] AS keywords

        MATCH (e)-[]->(a)
        WHERE 
        ALL(keyword IN keywords WHERE EXISTS {
            MATCH (e)-[:HAS_ATTRIBUTE]->(attr)
            WHERE NOT attr.name CONTAINS "简介"
                AND (attr.name CONTAINS keyword OR attr.value CONTAINS keyword)
        })
        AND ANY(keyword IN keywords WHERE a.name CONTAINS keyword OR a.value CONTAINS keyword)

        RETURN DISTINCT
            e.name AS head, 
            'HAS_ATTRIBUTE' AS relation,
            a.name + ":" + a.value AS tail,
            COALESCE(e.schema_type, "Unknown Node") AS head_props,
            COALESCE(a.schema_type, "Unknown Node") AS tail_props
```
