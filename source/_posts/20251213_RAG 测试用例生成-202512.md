# RAG 测试用例生成

## 0 背景

1. 验证 RAG 系统需要大量测试用例。
2. 人工收集成本高、耗时长。
3. 使用大模型自动生成问答对（QA）可以快速批量创建数据。

## 1 流程概览

**第一步：生成问答对（QA）**

1. 从原始 JSON 文件随机采样 N 条文档。
2. 使用大模型（Qwen3-32b）生成高质量问答对。
3. 写入 Excel 文件（`QA_ONLY_FILE`），格式：`问题` / `答案`。
4. 优势：
   - 数据和模型回答分离，方便多次 RAG 测试。
   - 可生成批量 QA，用于评测。

**第二步：RAG 回答验证**

1. 从 QA Excel 文件读取问题。
2. 使用 GraphRAG 系统回答问题，并获取上下文（retrieved chunks / triples）。
3. 写入 Excel 文件（`QA_RAGAS_FILE`），格式：`问题 / LLM答案 / RAG回答 / 上下文`。
4. 可以对比 LLM 生成答案与 RAG 回答，分析检索效果。

## 2 生成问答对

### 2.1 模块设计

#### 2.1.1 总体架构

**目标**：快速生成高质量 RAG 测试用例（QA），保存为 Excel，便于后续 RAG 系统评估。

**流程**：

> 原始 JSON → 数据采样 → LLM 生成 QA → 保存 Excel

#### 2.1.2 核心模块划分

| 模块            | 功能                                | 输入           | 输出                                    | 设计原则               |
| --------------- | ----------------------------------- | -------------- | --------------------------------------- | ---------------------- |
| **采样模块**    | 随机选择 N 条文档作为生成 QA 的基础 | 原始 JSON 文件 | 采样 JSON 数据                          | 单一职责，可复用       |
| **QA 生成模块** | 使用 LLM 根据文本生成单条问答对     | 文本内容       | QA dict：`{"question", "ground_truth"}` | 保证格式一致，异常处理 |
| **Prompt 模块** | 定义不同类型 QA 模板（v1/v2/v3）    | 文本内容       | LLM 输入 prompt                         | 可扩展不同 QA 类型     |
| **存储模块**    | 将 QA 数据写入 Excel 文件           | QA dict        | Excel 文件                              | 容错，逐条追加，防中断 |

#### 2.1.3 模块设计原则

1. **职责单一**：采样、生成 QA、存储 Excel 分开
2. **可复用**：模块独立，支持多次生成 QA 或扩展新流程
3. **接口统一**：QA 输出统一为字典格式 `{"question", "ground_truth"}`
4. **容错稳健**：生成失败可跳过，逐行写入 Excel
5. **可扩展**：可替换 LLM 模型或 Prompt，支持不同类型 QA

### 2.2 代码实现

```python
import json
import time
import requests
from openpyxl import Workbook, load_workbook
import os
import random

# ================== 配置 ==================
url = "http://10.0.0.228:9925/v1/chat/completions"
api_key = "sk-xxxxxxxxx"
model = "qwen3-32b"

HEADERS = {
    "Authorization": f"Bearer {api_key}",
    "Content-Type": "application/json",
}
# ================== QA 生成 Prompt ==================
# v1: 高质量综合问题，用于 RAG 系统评测
PROMPT_QA_SINGLE_v1 = """
你是一个专业的问答对数据集构建专家，擅长从资料中提炼“人类真实会问的问题”。

请根据【内容】生成【正好 1 条】高质量问答对，用于 RAG 系统评测。

【问题生成要求】
1. 问题应模拟真实用户的提问方式，而不是书面定义或标题复述
2. 优先生成需要“综合多句信息才能回答”的问题
3. 问题可以是原因、影响、作用、机制、条件、对比或结论类问题
4. 避免“是什么 / 有哪些 / 请介绍”这类浅层问题
5. 问题本身应具有独立意义，不依赖原文措辞

【答案生成要求】
1. 答案必须完全基于内容
2. 不引入内容之外的任何事实或推断
3. 表达清晰、完整，但不过度扩写

【输出要求】
- 只输出结果，不要解释
- 不要包含前言、总结或思考过程
- 返回格式必须严格为 JSON 数组，且长度必须为 1

【格式示例】
[
  {{
    "question": "问题",
    "ground_truth": "答案"
  }}
]

内容如下：
{content}
"""

# v2: 问题答案直接可在内容中找到
PROMPT_QA_SINGLE_v2 = """
你是一个用于构建 RAG 评测数据的问答对生成器。

请根据【内容】生成 1 条问答对，用于验证检索系统是否能找到相关文本。

要求：

【问题】
1. 使用真实用户的自然提问方式
2. 问题应能直接在内容中找到答案
3. 问题可以复述内容中的关键信息，但不要引入新概念

【答案】
1. 必须完全基于内容
2. 用自然、连贯的人类回答方式表述
3. 可以适度改写原文，但不得新增、推断或省略关键信息
4. 不要逐句照抄原文

【输出要求】
- 只输出结果，不要解释
- 不要包含任何前言或总结
- 返回格式必须严格为 JSON 数组，长度必须为 1

【格式示例】
[
  {{
    "question": "问题",
    "ground_truth": "答案"
  }}
]

内容如下：
{content}
"""

# v3: 枚举型问题，统计实体数量
PROMPT_QA_SINGLE_v3 = """
你是一个用于构建 RAG 评测数据 的问答对生成器。

请根据【内容】生成 1 条问答对，用于验证检索系统是否能够从某一共同属性、国家、类型、用途或范围出发，完整枚举相关实体，并正确统计数量。

【问题要求】

- 使用真实用户的自然提问方式，直接提问
- 不引用或指代文本本身（不得出现“根据以上内容”“文中提到”等表述）
- 问题必须从某一共同属性、国家、类型、用途或类别出发
- 问题应明确暗示需要列举多个对象，并体现需要统计数量
- 问题的范围必须完全受限于内容中实际出现的信息
- 不得引入内容之外的任何概念或实体

示例提问方式（仅作形式参考）：
- 某个国家一共有哪些型号的装备？分别是什么
- 某一类型的系统包含多少个实体？它们分别是哪些
- 某一类别下出现了哪几种型号
- 某类装备一共有多少种？具体包括哪些

【答案要求】

- 答案必须完全基于内容，不得补充常识或外部信息
- 使用自然语言的枚举形式回答，清晰列出所有符合条件的实体
- 不得遗漏内容中出现的任何符合条件的实体
- 不得解释、不评价、不总结规律
- 必须明确给出实体的总数量

【输出要求】

- 只输出结果，不要解释
- 不要包含任何前言或总结
- 返回格式必须严格为 JSON 数组
- 数组长度必须为 1

【格式示例】
[
  {{
    "question": "问题",
    "ground_truth": "答案"
  }}
]

内容如下：
{content}
"""

# ================== LLM 调用 ==================
def call_gpt(messages):
    """
    调用大模型接口生成文本
    Args:
        messages (list): GPT 聊天消息列表
    Returns:
        str: GPT 返回的文本内容
    """
    data = {
        "model": model,
        "messages": messages
    }
    resp = requests.post(url, headers=HEADERS, json=data, verify=False) # 发送 POST 请求
    resp.raise_for_status() # 如果请求失败抛出异常
    return resp.json()["choices"][0]["message"]["content"] # 返回 GPT 的回答内容

# ================== 生成问答 ==================
def generate_single_qa(text):
    """
    根据文本生成一条问答对
    Args:
        text (str): 文本内容
    Returns:
        dict or None: {"question": ..., "ground_truth": ...} 或 None（生成失败）
    """
    try:
        result = call_gpt([ # 调用 LLM 接口生成 QA，使用 v3 prompt
            {"role": "system", "content": PROMPT_QA_SINGLE_v3.format(content=text)}
        ])

        # 清理 think
        if "<think>" in result:
            result = result.split("</think>")[-1]

        qa_pairs = json.loads(result) # 将 GPT 输出解析为 JSON

        # 必须返回单条 dict
        if isinstance(qa_pairs, list) and len(qa_pairs) == 1:
            return qa_pairs[0]  # 这里改成返回 dict
        else:
            print("数量不符，跳过")
            return None

    except Exception as e:
        print("解析失败：", e)
        return None

# ================== JSON → 文本 ==================
def json_item_to_text(item: dict) -> str:
    """
    将 JSON 内容转换为可读文本，每个字段占一行
    Args:
        item (dict): JSON 对象
    Returns:
        str: 转换后的文本
    """
    lines = []
    for k, v in item.items():
        lines.append(f"{k}：{v}")
    return "\n".join(lines)

# ================== 写入 Excel ==================
def write_to_excel_append(qa_item, filename="./qa_only_result-v1.xlsx"):
    """
    将单条 QA 追加写入 Excel 文件
    Args:
        qa_item (dict): {"question": ..., "ground_truth": ...}
        filename (str): Excel 文件路径
    """    
    # 如果文件存在，打开并获取工作表
    if os.path.exists(filename):
        wb = load_workbook(filename)
        ws = wb.active
    else:
        wb = Workbook()
        ws = wb.active
        ws.title = "QA"
        ws.append(["问题", "答案"])
        
    # 写入一行数据
    ws.append([
        qa_item["question"],
        qa_item["ground_truth"],
    ])
    # 保存 Excel
    wb.save(filename)
    print(f"追加写入：{qa_item['question']}")

# ================== 配置参数 ==================
SAMPLE_SIZE = 100 # 采样数量
ORIGIN_FILE = "/data/hsw/youtu-graphrag-gb-2-fortest/hsw_test/results/military_v3_clean.json"
SAMPLE_FILE = ORIGIN_FILE + f"_sample{SAMPLE_SIZE}.json"
QA_ONLY_FILE = SAMPLE_FILE + "_qa_only_属性到实例.xlsx"

# ================== 主流程 ==================
def main():
    # 读取原始 JSON 数据
    with open(ORIGIN_FILE, "r", encoding="utf-8") as f:
        data = json.load(f)
        
    print(f"📚 原始文档数：{len(data)}")
    
    # 随机采样 SAMPLE_SIZE 条数据
    samples = random.sample(data, SAMPLE_SIZE)
    print(f"🎯 采样文档数：{len(samples)}")
    
    # 保存采样结果到文件，便于复用
    with open(SAMPLE_FILE, "w", encoding="utf-8") as f:
        json.dump(samples, f, ensure_ascii=False, indent=2)

    print(f"💾 随机采样已保存到: {SAMPLE_FILE}")

    # 重新读取采样文件（可选）
    with open(SAMPLE_FILE, "r", encoding="utf-8") as f:
        samples = json.load(f)
    
    success = 0
    # 遍历采样文本，生成 QA
    for idx, item in enumerate(samples, 1):
        print(f"\n[{idx}/{SAMPLE_SIZE}] 生成 QA 中...")

        text = json_item_to_text(item)
        if not text.strip(): # 如果文本为空，跳过
            continue

        qa = generate_single_qa(text)
        print(f"idx:{idx} qa:{qa}")
        if qa:
            write_to_excel_append(qa, QA_ONLY_FILE)
            success += 1
    print(f"\n🎉 完成：成功生成 {success} 条 RAGAS QA")

if __name__ == "__main__":
    main()
```

## 3 RAG 回答验证

### 3.1 模块设计

#### 3.1.1 总体架构

**目标**：读取 QA 测试用例（LLM 生成的问答对），使用 GraphRAG 系统获取答案及上下文，并保存到 Excel 文件，便于后续指标评估。

**流程**：

> QA Excel → 遍历问题 → GraphRAG 回答 + 检索上下文 → 保存 Excel

#### 3.1.2 核心模块划分

| 模块                  | 功能                                    | 输入                                                     | 输出                                         | 设计原则                            |
| --------------------- | --------------------------------------- | -------------------------------------------------------- | -------------------------------------------- | ----------------------------------- |
| **QA 读取模块**       | 从 Excel 文件读取 QA 问答对             | Excel 文件                                               | QA 列表 dict：`{"question", "ground_truth"}` | 单一职责，接口统一                  |
| **GraphRAG 问答模块** | 调用 GraphRAG API 获取答案和检索上下文  | 问题字符串                                               | dict：`{"answer", "retrieved"}`              | 封装 API 调用，保证返回格式一致     |
| **JSON→文本模块**     | 将 JSON 文档转换为可读文本              | JSON 对象                                                | 文本字符串                                   | 统一格式，便于 GraphRAG 或 LLM 使用 |
| **Excel 写入模块**    | 将单条 QA + GraphRAG 回答追加写入 Excel | dict：`{"question","ground_truth","answer","retrieved"}` | Excel 文件                                   | 容错、逐行追加，防中断              |

#### 3.1.3 模块设计原则

1. 职责单一：读取、调用 GraphRAG、写入 Excel 分开

2. 接口统一：GraphRAG 返回固定结构 {"answer","retrieved"}

3. 容错稳健：逐行写入，单条失败不影响整体

4. 可扩展：可替换 GraphRAG API 或扩展其他检索系统

### 3.2 代码实现

```python
import os
import json
import time
import random
import requests
from typing import List, Dict, Union
from openpyxl import Workbook, load_workbook

# ================== 配置 ==================
url = "http://10.0.0.228:9925/v1/chat/completions"
api_key = "sk-xxxxxxxxx"
model = "qwen3-32b"

HEADERS = {
    "Authorization": f"Bearer {api_key}",
    "Content-Type": "application/json",
}


# ================== JSON → 文本 ==================
def json_item_to_text(item: dict) -> str:
    lines = []
    for k, v in item.items():
        lines.append(f"{k}：{v}")
    return "\n".join(lines)

def read_qa_from_excel(filename: str, only_questions: bool = False) -> Union[List[str], List[Dict[str, str]]]:
    """
    从 Excel 文件读取问答对
    Args:
        filename: Excel 文件路径
        only_questions: True 只返回问题列表，False 返回 [{'question': ..., 'ground_truth': ...}, ...]
    Returns:
        List[str] 或 List[Dict[str, str]]
    """
    if not os.path.exists(filename):
        print(f"文件不存在：{filename}")
        return []

    wb = load_workbook(filename)
    ws = wb.active

    results = []
    for row in ws.iter_rows(min_row=2, values_only=True):  # 跳过表头
        question = row[0]
        answer = row[1] if len(row) > 1 else ""
        if not question:
            continue
        if only_questions:
            results.append(question)
        else:
            results.append({"question": question, "ground_truth": answer})
    
    print(f"✅ 从 Excel 读取 {len(results)} 条 {'问题' if only_questions else '问答对'}")
    return results

# ================== GraphRAG 问答 ==================
def ask_graphrag(question):
    """
    使用 GraphRAG API 获取问题回答及相关上下文
    
    Args:
        question: 待提问问题字符串
    Returns:
        answer: GraphRAG 生成的答案
        retrieved: 检索到的上下文（chunks 和 triples）
    """
    url = "http://0.0.0.0:8014/api/ask-question"
    payload = {
        "dataset_name": "military_v3_clean",  # 使用的数据集
        "question": question
    }
    params = {"client_id": "test_client"} # 客户端标识，可自定义

    resp = requests.post(url, json=payload, params=params)
    resp_json = resp.json()
    # 将检索到的上下文格式化成字符串
    retrieved = f"【retrieved_chunks：{resp_json.get('retrieved_chunks', [])}】 【retrieved_triples:{resp_json.get('retrieved_triples', [])}】"
    return resp_json["answer"], retrieved

def list_to_str(data):
    """多层列表扁平化为字符串"""
    if isinstance(data, list):
        return ", ".join(list_to_str(i) for i in data)
    return str(data)

def write_to_excel_append(qa_item, filename="./qa_rags_result-v1.xlsx"):
    """
    单条 QA 数据追加写入 Excel 文件
    Args:
        qa_item: dict 类型，包含 question, ground_truth, answer, retrieved
        filename: Excel 文件路径
    """
    if os.path.exists(filename):
        wb = load_workbook(filename)
        ws = wb.active
    else:
        wb = Workbook()
        ws = wb.active
        ws.title = "QA"
        ws.append(["问题", "答案", "回答", "上下文"])

    retrieved_str = list_to_str(qa_item["retrieved"])
    ws.append([
        qa_item["question"],
        qa_item["ground_truth"],
        qa_item["answer"],
        retrieved_str
    ])
    wb.save(filename)
    print(f"追加写入：{qa_item['question']}")

# ================== 文件路径配置 ==================
from gen_qa_new_v3_qaOnly import SAMPLE_SIZE, SAMPLE_FILE, QA_ONLY_FILE
QA_ONLY_FILE = "/data/hsw/youtu-graphrag-gb-2-fortest/hsw_test/military_v3_clean.json_sample100.json_qa_only_属性到实例.xlsx"
QA_RAGAS_FILE = "/data/hsw/youtu-graphrag-gb-2-fortest/hsw_test/military_v3_clean.json_sample100.json_rags_属性到实例.xlsx"  # SAMPLE_FILE + "_qa_rags.xlsx"

# ================== 主流程 ==================
def main():
    # 读取 JSON 样本文件
    with open(SAMPLE_FILE, "r", encoding="utf-8") as f:
        samples = json.load(f)
    print(f"导入文档数：{len(samples)}")

    # 从 Excel 读取问答对
    qa = read_qa_from_excel(QA_ONLY_FILE) # 从xlsx读入问答对（取问题）
    
    success = 0
    # 遍历样本生成 GraphRAG 回答
    # for idx, item in enumerate(samples[38:], 39): #(samples, 1): # 从39个开始
    for idx, item in enumerate(samples, 1): # 从1开始

        print(f"\n[{idx}/{SAMPLE_SIZE}] 生成 graphrag 回答和上下文 中...")

        text = json_item_to_text(item)
        if not text.strip(): # 去掉字符串两端的空白字符，判断是否是空字符串
            continue

        question = qa[idx - 1]["question"]
        ground_truth = qa[idx - 1]["ground_truth"]

        # 获取 GraphRAG 回答和上下文
        answer, retrieved = ask_graphrag(question)
        qa_item = {
            "question": question,
            "ground_truth": ground_truth,    # LLM 生成答案
            "answer": answer,       	 	 # GraphRAG 回答
            "retrieved": retrieved           # GraphRAG 上下文
        }
        # 写入 Excel
        write_to_excel_append(qa_item, QA_RAGAS_FILE)

    print(f"\n🎉 完成：成功生成 {success} 条 RAGAS QA")

if __name__ == "__main__":
    main()
```
